# 单词完整记忆逻辑

## 1. 学习流程
1. **获取学习计划**：通过 `/vocabulary/plan` API 获取今日学习计划，包括新单词数量和复习单词数量
2. **开始学习**：点击"开始学习"按钮，通过 `/vocabulary/start` API 获取学习单词列表
3. **学习单词**：逐个学习单词，包括认识阶段和记忆检测阶段
4. **提交答案**：通过 `/vocabulary/check` API 提交答案，系统会更新学习记录

## 2. 记忆算法
系统采用艾宾浩斯遗忘曲线算法，具体实现如下：

### 2.1 复习间隔
- 第1次复习：1天后
- 第2次复习：1天后（共2天）
- 第3次复习：3天后（共5天）
- 第4次复习：7天后（共12天）
- 第5次复习：15天后（共27天）
- 第6次复习：30天后（共57天）

### 2.2 状态转换
1. **新单词**：初始状态为 `learning`，`review_stage` 为 0
2. **学习过程**：
   - 答对：`correct_count` +1，`review_stage` +1
   - 答错：`wrong_count` +1，`review_stage` 重置为 0，第二天重新复习
3. **掌握标准**：当 `review_stage` >= 6 时，状态变为 `mastered`，认为单词已掌握

### 2.3 复习机制
- 系统通过 `GetDueReviewVocabularies` 函数获取需要复习的单词
- 复习单词的条件是 `next_review_date <= CURRENT_DATE`
- 每次复习后，系统会根据答题情况更新 `next_review_date`

## 3. 数据模型
- **ab_vocabulary**：单词表，存储单词基本信息
- **ab_learning_record**：学习记录表，存储用户的学习进度和状态
  - `status`：状态（new, learning, mastered, reviewing）
  - `review_stage`：复习阶段（0-5）
  - `next_review_date`：下次复习日期
  - `correct_count`：正确次数
  - `wrong_count`：错误次数

## 4. 每日学习限制
- 每日新单词学习数量限制为5个
- 复习单词数量不限制
- 系统会记录用户每天学习的新单词数量，确保不超过限制

## 5. 学习记录更新流程
1. 新单词学习时，创建学习记录，状态为 `learning`
2. 每次答题后，更新学习记录：
   - 答对：增加正确次数，提升复习阶段，计算下次复习时间
   - 答错：增加错误次数，重置复习阶段，设置第二天复习
3. 当复习阶段达到6时，标记为已掌握

## 6. 前端实现
- 学习模式：支持正常模式、默英文模式、默中文模式
- 检测方式：支持英文到中文、中文到英文、听力检测
- 学习统计：显示累计学习、今日学习、已掌握、连续学习、正确率等统计数据

## 7. 后端API
- `/vocabulary/plan`：获取今日学习计划
- `/vocabulary/start`：开始学习，获取学习单词列表
- `/vocabulary/check`：提交答案，更新学习记录
- `/vocabulary/options`：获取词汇选项（用于选择题）

这套记忆逻辑基于艾宾浩斯遗忘曲线，通过间隔重复的方式帮助用户更有效地记忆单词，同时通过状态管理和复习机制确保学习效果。
